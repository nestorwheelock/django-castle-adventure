INTERACTIONS:

Choice Buttons [A] [B] [C] [D]:
- Large click/touch targets (minimum 48px height)
- Keyboard: Press A, B, C, or D key to select
- Hover: Background changes to #b22222 (lighter red)
- Click: Navigates to next scene (page reload or AJAX)

Locked Choice [D]:
- Grayed out with lock icon 🔒
- Tooltip on hover: "Magical barrier blocks this path"
- Not clickable
- Screen reader: "Option D: Try stairs. Locked. Requires magical key."

[💾 Saved] Indicator:
- Shows auto-save status
- Animates briefly after each choice: "Saving..." → "Saved ✓"
- Click to manually save (redundant, but confirms)

[☰ Menu] Button:
- Opens dropdown menu:
  - Save Game
  - Load Game
  - Settings (accessibility, text size)
  - How to Play
  - Quit to Main Menu

[View Full] Inventory Link:
- Opens full inventory modal (see 03-inventory-view.txt)
- Shows all items with descriptions
- Keyboard: Press 'I' key

DESIGN NOTES:

Layout:
- ASCII art section: 20% of height
- Scene description: 30% of height
- Choices: 40% of height
- Inventory bar: 10% of height

Colors:
- Scene description background: #2a2a2a (slightly lighter than main)
- Choice buttons: #8b0000 border, #1a1a1a fill
- Choice hover: #b22222 background
- Locked choices: #555 (gray)
- ASCII art: #00ff00 (terminal green) on #0a0a0a

Typography:
- Scene title: 24px, bold, sans-serif
- Scene description: 18px, serif (Georgia)
- Choices: 16px, sans-serif
- Inventory items: 14px

Accessibility:
- Tab order: Choice A → B → C → D → Inventory → Menu
- Arrow keys navigate choices
- Enter/Space selects choice
- Screen reader announces: "Scene 7, Castle Entrance Hall. Description: [reads text]. 4 choices available."

───────────────────────────────────────────────────────────────────────────────────

RESPONSIVE BEHAVIOR:

Desktop (> 1024px):
- Full ASCII art display (20 lines)
- Side-by-side layout possible

Tablet (768-1024px):
- Slightly smaller ASCII art (15 lines)
- Stacked vertical layout

Mobile (< 768px):
- Simplified ASCII art (10 lines) or text-only description
- Large choice buttons (60px height for touch)
- Collapsible inventory (tap to expand)

───────────────────────────────────────────────────────────────────────────────────

EDGE CASES:

1. Very long scene description (>500 words):
   - Add scrollable container
   - "Read more" expand/collapse

2. Many choices (>6):
   - Paginate or group logically
   - Indicate "More options below"

3. Slow connection:
   - Show loading spinner after click
   - Disable button to prevent double-submit

4. JavaScript disabled:
   - Form submission with POST
   - Full page reload (still functional)
